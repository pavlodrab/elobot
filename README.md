# ⚽ FC Mobile League Bot

Telegram-бот для автоматического ведения лиги FC Mobile: ELO-рейтинг,
групповые турниры, плей-офф, статистика бомбардиров, уведомления **и
автоматическое распознавание скринов матчей** (OCR).

Поддерживает два типа турниров: **ВСА** (где много голов) и **РИ** (где голов мало).

---

## 🚀 Быстрый старт

### 1. Создай бота в Telegram

1. Открой [@BotFather](https://t.me/BotFather)
2. `/newbot` → введи имя и username
3. Скопируй токен (`BOT_TOKEN`)

### 2. Узнай свой Telegram ID

Напиши [@userinfobot](https://t.me/userinfobot) — он пришлёт твой `id` (это твой `ADMIN_ID`).

### 3. Запуск через Docker (рекомендуется)

```bash
git clone <repo> fc_league_bot
cd fc_league_bot

cat > .env <<EOF
BOT_TOKEN=1234567890:AAABBBCCC...
ADMIN_IDS=123456789,987654321
EOF

docker-compose up -d
docker-compose logs -f
```

Docker-образ автоматически ставит `tesseract` + русский/английский
языковые пакеты — OCR работает «из коробки».

### 4. Запуск без Docker

```bash
# Установка системных зависимостей (нужен tesseract для OCR)
sudo apt-get install -y tesseract-ocr tesseract-ocr-rus tesseract-ocr-eng

cd fc_league_bot
pip install -r requirements.txt

export BOT_TOKEN="твой_токен"
export ADMIN_IDS="твой_telegram_id"

python bot.py
```

## 🆕 Что нового в v13

- 🖼 **Кастомный фон для PNG-картинок турнира.** Создатель / админ
  турнира может прислать любое фото и сделать его фоном `/standings` и
  `/playoff`:
  - `/set_tournament_bg [ID]` — пришли фото с такой подписью **или**
    ответь этой командой на любое фото в чате. Бот скачает картинку,
    положит её в `tournament_bg/<tid>.jpg` и сохранит путь в
    `tournaments.bg_image_path`. Поверх фото на рендере накладывается
    лёгкий тёмный overlay, чтобы текст оставался читаемым на любом
    изображении.
  - `/clear_tournament_bg [ID]` — сбросить фон, картинки снова на
    стандартном тёмном фоне.
- 🚀 **Кнопка ручного перехода стадии плей-офф.** Под `/playoff` теперь
  рисуется кнопка **«🚀 Перейти к следующей стадии»**. Видна только
  тем, кто может управлять турниром (создатель / делегированные админы /
  root) и только когда есть что двигать (текущая стадия закрыта,
  следующая ещё не сгенерирована). Тыкаешь — бот вызывает тот же путь,
  что `/advance_playoff` (создаёт следующую стадию + анонс + лички
  участникам). Полезно для турниров, начатых до v11 (без авто-перехода).
- 🩹 **Фикс реального бага продвижения стадии для 2-leg формата.**
  В предыдущих версиях бот мог решить, что стадия плей-офф ещё не
  закрыта, если в БД оставались phantom-строки (старые `pending`
  дубликаты для той же пары/leg от прежних версий). Теперь
  `advance_playoff` дедуплит phantom-строки по `(пара, стадия, leg)`,
  выбирая авторитетную (`confirmed > reported > pending`, при равном
  статусе — самую новую). И `/advance_playoff` теперь доступен не
  только root-админам, но и создателю турнира + делегированным
  tournament-админам.
- 🛟 **Самовосстановление поломанной `stage`.** Если в БД от старого
  бота осталось рассинхрон состояний (бракет создан, а
  `tournaments.stage` всё ещё `groups`), `_maybe_auto_advance` теперь
  чинит `stage` в `playoff` сам и двигает турнир дальше.

## 🆕 Что нового в v12

- 🤝 **`/h2h @a @b`** — сводка личных встреч между двумя игроками во
  всех турнирах: сколько матчей, кто сколько раз победил, голы, последние
  5 встреч. С разбивкой по типу турнира (ВСА / РИ). Если указан только
  один игрок (`/h2h @opponent`) — сравнение с тобой.
- 📅 **`/my_deadlines`** — все мои незакрытые матчи отсортированные
  по дедлайну, с обратным отсчётом ("через 3ч 12м" / "просрочено 5ч").
  Дублируется кнопкой «📅 Мои матчи / дедлайны» прямо в `/profile`.
- 📜 **`/tlog [ID] [N]`** — журнал турнира: кто и когда добавил
  игрока, провёл walkover, поменял описание/канал, продвинул стадию,
  назначил/снял админа, сделал рассылку, withdraw. Доступно только
  админам этого турнира. По умолчанию последние 30 записей; можно
  явно (`/tlog 7 50`).
- 🔮 **`/playoff_preview [ID]`** — превью пар плей-офф **до** того
  как `/start_playoff` зафиксирует сетку. Использует тот же алгоритм
  жеребьёвки (cross-group: 1A vs 2B, 1B vs 2A …), что и
  `generate_playoff` — поэтому показывает реальный результат при
  текущих очках.
- ⚠️ **`/withdraw [ID] @user`** — снять игрока из плей-офф/группы.
  Все его незакрытые матчи в этом турнире проводятся через
  техническое поражение (0:3 в пользу соперника), запись попадает в
  `/tlog`, после чего бот автоматически продвигает стадию и оповещает
  оставшихся участников. Удобно при отвале по ходу серии.
- 📣 **`/broadcast [ID] <текст>`** — рассылка всем участникам турнира
  в личку (можно ответом этой командой на любое сообщение —
  тогда разошлётся текст того сообщения). Доступно создателю /
  делегированным админам / root. Запись в `/tlog`.

## 🆕 Что нового в v11

- 📸 **Несколько скринов одного матча**: если в скриншоте не помещаются
  все бомбардиры, отправь второе фото в **том же альбоме Telegram**
  (выделенные сразу друг за другом фото). Бот сравнит счёт и команды
  с уже распознанным фото и **дополнит список голов** этого матча,
  не создавая дубликат. Ответит «📎 Это продолжение того же матча.
  Добавил N новых голов».
- 🚀 **Авто-переход стадий + уведомления**: после того как все пары
  очередной стадии плей-офф (1/8 → 1/4 → 1/2 → финал) подтверждены,
  бот сам создаёт следующую стадию. В чат турнира улетает анонс
  с парами и дедлайном, всем участникам новой стадии — личка
  с описанием их матча. Работает во всех путях завершения матча:
  авто-подтверждение, подтверждение админом, бул-волковер, дедлайн,
  /admin_report, /simulate.
- 🛡 **Привязка админов к турниру**: теперь редактировать турнир может
  только главный админ из `ADMIN_IDS`, **создатель турнира** и
  **админы конкретного турнира**, которых может назначать создатель
  (или главный админ). Команды `/grant_admin` дают только бот-админ
  права (баны, /elo, /admin_report) — но **не дают** автоматически
  возможности редактировать чужие турниры.
  - `/add_tadmin [ID] @user [заметка]` — назначить админа турнира
  - `/remove_tadmin [ID] @user` — снять
  - `/tadmins [ID]` — список
- 🪜 **Полная сетка плей-офф**: `/playoff` (картинка) и
  `format_playoff_bracket` (текст) теперь рисуют **весь** путь до
  финала, начиная от текущей стадии — недостающие стадии
  показываются плейсхолдерами «TBD». Например, если идёт 1/2,
  справа всё равно появится колонка «Финал» с одним TBD-cardом.

## 🆕 Что нового

- 🎮 **Игровые ники**: каждый игрок задаёт свой ник в игре (`/setnick`),
  бот по нему сопоставляет скрин с профилем.
- 🏟 **ВСА / РИ**: турниры теперь делятся на два типа.
  Создавай так: `/create_tournament Сезон1 вса` или `... ри`.
  Можно одновременно вести произвольное количество ВСА- и РИ-турниров;
  команды без явного ID берут самый свежий турнир нужного типа, либо
  турнир, к которому привязан текущий чат (`/bind_tournament <ID>`).
- 👥 **Игроки сами создают турниры** — больше не нужно быть админом.
  Создатель турнира получает права на `/add_player`, `/start_tournament`,
  `/start_playoff` для своего турнира.
- 🏅 **Стартовый ELO = 0** (раньше было 1000). Ранги пересчитаны:
  Bronze < 50 ≤ Silver < 150 ≤ Gold < 250 ≤ Diamond < 400 ≤ Legendary.
- 📸 **Распознавание скринов матчей**: просто пришли фото скрина матча —
  бот сам прочитает счёт, поймёт ВСА это или РИ (по плашке лиги),
  найдёт соперника по игровому нику и предложит подтвердить.
- 🚫 **Баны**: админ может временно/бессрочно отстранять игроков.
  Команды `/ban`, `/unban`, `/banned`. Забаненный не сможет джойниться
  в турниры, репортить или слать скрины.
- 📝 **Описание турнира**: создатель задаёт `/set_description`. Текст
  показывается в `/tournaments`.
- 🔗 **Условие подписки на канал**: создатель ставит `/set_channel @ch` —
  и для участия в турнире нужно быть подписанным. Бот должен быть
  добавлен в канал (можно без админ-прав, чтобы видеть участников).
  Снять — `/clear_channel`.
- ⚖️ **Ручная правка ELO админом**: `/elo @user +50` (дельта) или
  `/setelo @user 200` (абсолют). Каждое изменение логируется в профиле.
- 🐞 **Баги / предложения**: кнопка под `/start` или команда `/feedback` —
  сообщение (с фото при желании) пересылается в личку всем админам.
- 🛂 **Подтверждение матчей админом**: после того как соперник
  подтвердил результат, матч уходит админу в личку с кнопками
  `[✅ Засчитать] [❌ Отклонить]`. ELO меняется только после согласия.
- 🌐 **Общий пул ELO** — для официальных турниров (создаёт админ из
  `ADMIN_IDS`). `/top` / `/top_vsa` / `/top_ri` считают только их.
- 🏠 **Изолированный ELO для турниров игроков**: турнир, созданный
  обычным игроком, имеет собственный лидерборд. Общий ELO/ВСА/РИ
  у участников **не задеваются** — рейтинг живёт только внутри турнира.
  Смотреть: `/leaderboard [ID|вса|ри]`.
- 🪟 **Меню**: после `/start` снизу всегда висит главное меню
  (Турниры, Профиль, Репорт, Топ, Настройки, Связь, Админ для админа,
  Помощь). Большинство флоу работают через тапы кнопок и пошаговые
  визарды. Команды по-прежнему работают параллельно.

## 🏆 Как провести турнир

### Шаг 1: Зарегистрироваться и указать ник в игре
```
/register
/setnick MyInGameNickname
```

### Шаг 2: Создать турнир
```
/create_tournament Сезон1 вса
/create_tournament Зимний РИ ри

# С опцией авто-техпоражения при просрочке дедлайна:
/create_tournament Сезон1 вса тех          # → 0:3 для player1, 3:0 для player2
/create_tournament Сезон1 вса тех 0:0      # → ничья 0:0 при просрочке (для обоих)
/create_tournament Сезон1 вса 4 6 тех 0:0  # 4 группы по 6, авто-ТП ничьёй

# С явным числом групп / игроков на группу:
/create_tournament Сезон1 вса 4            # 4 группы (бот сам распределит игроков)
/create_tournament Сезон1 вса 4 6          # 4 группы по 6 игроков
```

### Шаг 3: Добавить игроков (создатель турнира)
```
# По одному:
/add_player @player1
/add_player @player2

# Или сразу пачкой через запятую/пробел:
/add_player @player1, @player2, @player3
/add_player @a @b @c ри
```

### Шаг 4: Запустить жеребьёвку
```
/start_tournament
```
Если активны и ВСА и РИ — укажи: `/start_tournament вса`.

### Шаг 5: Игроки репортуют результаты

#### Вариант А — Фото скрина (новое!)
Просто пришли в чат бота фото скрина матча.
Бот распознает счёт, тип турнира и попросит подтвердить.

В подписи к фото можно явно указать тип, если плашку не распознало:
```
вса
```
или
```
ри
```

#### Вариант Б — Текстом
```
/report 3:2 @opponent
/report 3:2 @opponent вса
```

Соперник подтверждает кнопкой ✅ или командой `/confirm`.

### Шаг 6: Плей-офф
```
/start_playoff
```
Бот автоматически выбирает лучших из каждой группы и строит сетку.

| `/dispute` | Оспорить результат |

| `/profile [@user]` | Профиль игрока |
| `/matches` | Твои последние матчи |
| 📸 *фото скрина* | Авто-распознавание счёта и турнира |
| `/feedback <текст>` | Отправить баг/предложение всем админам |
| `/cancel` | Отменить ожидание фидбека |

### Для создателя турнира

| Команда | Описание |
|---------|----------|
| `/add_player @user1[, @user2, ...] [вса\|ри]` | Добавить одного или нескольких игроков в свой турнир (запятые/пробелы — по вкусу) |
| `/list_players [ID]` | Состав турнира (без ID — привязанный к чату или активный) |
| `/replace_player @old @new [ID]` | Заменить игрока в активном турнире, не теряя расписание (подтверждённые матчи остаются за `@old`) |
| `/set_description <текст>` | Описание/правила своего турнира |
| `/set_channel @channel` | Требовать подписку на канал (бот должен быть в канале) |
| `/clear_channel` | Снять условие подписки |
| `/start_tournament [вса\|ри]` | Жеребьёвка и старт |
| `/start_playoff [ID\|вса\|ри]` | Запустить плей-офф (идемпотентно — повторный вызов не плодит дубли) |
| `/finish_tournament [ID]` | Завершить турнир (с подтверждением). Без ID — берёт активный турнир этого чата. Также есть кнопка 🏁 в `/tournaments` |

### Только для админа (`ADMIN_IDS` или promoted)

| Команда | Описание |
|---------|----------|
| `/ban @user [длительность] [причина]` | Бан: `24` (часы), `7d`, `30m`, `perm` |
| `/unban @user` | Снять бан |
| `/banned` | Список забаненных (доступно всем) |
| `/elo @user +50 [причина]` | Изменить ELO на дельту |
| `/setelo @user 200 [причина]` | Задать абсолютное значение ELO |
| `/walkover @user` | Засчитать тех. поражение |
| `/grant_admin @user [коммент]` | Выдать админку (или ответом на сообщение, или по ID) |
| `/revoke_admin @user` | Снять админку (root-админов из `ADMIN_IDS` снять нельзя) |
| `/admins` | Список текущих админов |
| `/admin_setnick @user InGameNickname` | Задать/поменять игровой ник любому игроку (или ответом на сообщение, или по ID). Если игрока ещё нет в БД — он создастся с этим ником. |
| `/admin_report @u1 @u2 score1:score2 [ID]` | Внести результат за игроков, минуя подтверждение соперника. ELO считается, плей-офф автоматически продвигается. Если пара уже сыграла max матчей (по настройкам) — откажет. |
| `/edit_match #<ID> score1:score2` | Перезаписать счёт уже подтверждённого матча: откатывает ELO/статистику и пересчитывает с новым счётом. |
| `/delete_match #<ID>` | Полностью удалить матч: откатить ELO/статистику (если был processed), удалить голы и запись. Для лишних/дублей. |
| `/simulate [ID]` | Авто-сыграть все оставшиеся матчи турнира. Счета подбираются с весом по ELO (фаворит чаще выигрывает). ELO/W-D-L и лидерборды обновляются как обычно. С подтверждением. Без ID — берёт активный турнир этого чата. |
| `/set_playoff_slots <ID> <N>` | Сколько игроков из каждой группы выходит в плей-офф (1–8). По умолчанию 2. |
| `/advance [ID]` | Принудительно продвинуть турнир: построить bracket после групп, или перейти на следующий раунд плей-офф. Обычно срабатывает автоматически после каждого матча. |
| `/walkover @loser [@winner] [ID]` | Техническое поражение 0:3. Без `@winner` берётся последний pending-матч проигравшего. |

> **Иерархия админов:**
> - **Root-админы** — те, чей `telegram_id` указан в env-переменной `ADMIN_IDS`. Снять/изменить можно только перезапуском бота с другим `ADMIN_IDS`.
> - **Runtime-админы** — те, кому root-админ выдал админку через `/grant_admin`. Хранятся в БД, снимаются через `/revoke_admin`. Имеют те же права на создание официальных турниров и админ-команды.

## ⚙️ Как работает ELO

- Стартовый рейтинг: **0** (раньше 1000)
- Минимум: **−500**
- K-фактор: **32** (для новичков — 48 в первых 10 матчах)
- Множитель за разницу голов: ×1.0 до ×1.5

### Два пула рейтинга

- **Общий пул** (`players.elo` / `elo_vsa` / `elo_ri`) — обновляется
  только матчами в **официальных** турнирах (`tournaments.is_official = 1`).
  Это все турниры, созданные админами из `ADMIN_IDS`. Команды:
  `/top`, `/top_vsa`, `/top_ri`.
- **Локальный (изолированный) пул** (`tournament_elo` на каждый
  турнир) — обновляется матчами в **турнирах игроков**
  (`tournaments.is_official = 0`). Эти матчи **не влияют** на
  `players.elo` / `elo_vsa` / `elo_ri`. Команда: `/leaderboard [ID|вса|ри]`.
- В обоих режимах глобальные статы (`wins/losses/draws`, голы, серии,
  сухие матчи) обновляются — изолируется только рейтинг.
- Существующие турниры на момент миграции получают
  `is_official = 1`, чтобы их прошлые матчи не влияли «задним числом».

## 📸 Как работает распознавание скринов

1. Игрок присылает в чат фото скрина результата матча.
2. Бот через `tesseract` (OCR) читает фиксированные участки
   изображения (см. `ocr.py`):
   - центральный счёт ("1 - 1")
   - имена команд / игроков слева и справа
   - плашку лиги ("Лига Гвардиольыча", "RI", "VSA", ...)
3. По плашке определяется тип турнира (ВСА / РИ). Если не получилось —
   тип берётся из подписи к фото; если и там нет — бот спрашивает кнопками.
4. Имя соперника со скрина сопоставляется с зарегистрированными
   `game_nickname` через нечёткое сравнение (`difflib.SequenceMatcher`).
5. Бот показывает что распознал, ты подтверждаешь — отправляется
   стандартный репорт, который соперник должен подтвердить.

Если OCR ошибся — всегда есть текстовый фолбэк:
`/report 3:2 @opponent вса`.

## 🛠 Миграция со старой версии

Бот сам мигрирует БД при старте (через `init_db()`):
- добавит колонки `game_nickname`, `tournament_type`, `created_by`,
  `stats_extra` если их ещё нет.
- добавит колонку `tournaments.is_official` (по умолчанию `1`) и
  таблицу `tournament_elo` для изолированных лидербордов
  турниров игроков.
- старые турниры по умолчанию помечаются как `vsa` (можешь поменять
  через SQL вручную).
- ELO **существующих** игроков НЕ сбрасывается. Если хочешь сбросить
  всех в 0, выполни в БД:
  ```sql
  UPDATE players SET elo = 0;
  ```
- Если хочешь, чтобы какой-то конкретный старый турнир стал
  «локальным» (общий ELO не зачитывается), выставь
  `is_official = 0` руками:
  ```sql
  UPDATE tournaments SET is_official = 0 WHERE id = <ID>;
  ```
  Замечание: прошлые матчи этого турнира уже учтены в общем ELO —
  миграция не «откатывает» их обратно.

## 🚂 Деплой на Railway (Postgres)

Бот умеет работать на двух базах:

- **SQLite** (по умолчанию, для локалки) — путь к файлу из `DB_PATH`
  (по умолчанию `./league.db`).
- **Postgres** (рекомендуется на Railway) — включается автоматически, если
  установлена переменная окружения `DATABASE_URL`. Railway-плагин Postgres
  её прописывает сам.

Шаги:

1. Создай проект на [railway.app](https://railway.app) и подключи свой
   git-репозиторий с этим кодом.
2. В разделе **+ New → Database → Add Postgres** добавь Postgres-плагин.
   Railway сам прокинет `DATABASE_URL` в окружение сервиса.
3. В разделе **Variables** твоего сервиса задай:
   - `BOT_TOKEN` — токен от `@BotFather`
   - `ADMIN_IDS` — список Telegram ID через запятую
   - (опционально) пере-определи `DATABASE_URL`, если хочешь свой Postgres
4. В корне репозитория уже лежат:
   - `Procfile` — `worker: python bot.py` (Railway/Heroku style)
   - `runtime.txt` — `python-3.11`
   - `nixpacks.toml` — ставит `tesseract-ocr` (+ rus/eng) на сборке
   - `requirements.txt` — включая `psycopg2-binary` для Postgres
5. Деплой стартанёт автоматически. Лог будет показывать «init_db OK» —
   значит таблицы создались/обновились.

Файлы Dockerfile / docker-compose.yml оставлены для локального запуска
со SQLite. Они не нужны на Railway, но и не мешают.

### Привязка чата к турниру

Чтобы бот в группе/канале засчитывал скрины **только** в нужный турнир:

- Создай турнир прямо из этой группы:
  `/create_tournament Сезон1 вса` — чат привязывается автоматически.
- Или привяжи существующий: `/bind_tournament 27` (нужно быть создателем
  турнира или админом бота). Отвязать: `/unbind_tournament`.
- Альтернатива — указать турнир в подписи к фото:
  - `#турнир 27` или `#27`
  - `тур 27`, `tournament 27`, `tour 27`
  - подстрока имени турнира — например, для турнира **«Тур Гвардиолыча»**
    подойдёт фото с подписью `Гвардиолыча`.

Если ни чат не привязан, ни в подписи нет ссылки на турнир — бот **молча
игнорирует** фото (никаких ответов, никаких ошибок). Это сделано чтобы
случайные скрины в общих чатах не засоряли бота.

## ❓ FAQ

**Q: Бот не распознаёт мой ник на скрине, что делать?**
A: Убедись что ты задал свой игровой ник через `/setnick`, и что ник
на скрине совпадает (хотя бы частично). При неуверенности бот
предложит выбрать соперника кнопками. Текстовый фолбэк всегда работает:
`/report 3:2 @opponent вса`.

**Q: Можно ли вести несколько турниров одновременно?**
A: Да — произвольное количество ВСА- и РИ-турниров. Команды
`/table`, `/playoff`, `/start_tournament`, `/start_playoff` принимают
опциональный ID или ключевое слово `вса` / `ри`. Без аргумента:
сначала ищется турнир, привязанный к этому чату (`/bind_tournament <ID>`),
потом — самый свежий активный (с учётом типа, если указан); при
нескольких активных и `/table` / `/playoff` без аргумента покажется
кнопочный селектор.

**Q: Что если игрок не зарегистрирован, но я его добавляю в турнир?**
A: Бот создаёт профиль автоматически. Игрок может зарегистрироваться
позже командой `/register`, и его telegram_id привяжется к профилю.

**Q: Как оспорить результат?**
A: Нажать кнопку «❌ Оспорить» в DM или вызвать `/dispute`.
Матч возвращается в статус `pending`, организатор решает вручную.
